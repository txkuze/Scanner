# Security and Legal Notice

## Important Legal Information

### Authorization Required

This vulnerability scanning tool MUST ONLY be used on:

1. **Systems you own** - Websites, servers, and infrastructure under your direct ownership
2. **Systems you have explicit written authorization to test** - Valid pentesting contracts or bug bounty programs
3. **Educational environments** - Authorized training labs and CTF competitions

### Illegal Activities

Unauthorized vulnerability scanning is ILLEGAL in most jurisdictions and may violate:

- Computer Fraud and Abuse Act (CFAA) in the United States
- Computer Misuse Act in the United Kingdom
- Similar laws in other countries
- Terms of Service of cloud providers and hosting companies

**Penalties may include:**
- Criminal prosecution
- Civil lawsuits
- Substantial fines
- Imprisonment

## Ethical Use Guidelines

### DO:
- Obtain explicit written permission before scanning any system
- Use this tool for defensive security and awareness
- Report vulnerabilities responsibly through proper channels
- Respect rate limits and avoid causing service disruption
- Keep credentials and findings confidential
- Use results to improve security posture

### DON'T:
- Scan systems without authorization
- Share vulnerabilities publicly before they are fixed
- Use findings for malicious purposes
- Attempt to exploit discovered vulnerabilities
- Perform denial-of-service attacks
- Access unauthorized data or systems

## Responsible Disclosure

If you discover vulnerabilities using this tool:

1. **Notify the owner immediately** through appropriate security contacts
2. **Provide sufficient detail** for them to reproduce and fix the issue
3. **Give reasonable time** for remediation (typically 90 days)
4. **Do not disclose publicly** until the issue is resolved
5. **Follow the organization's** vulnerability disclosure policy if available

## Tool Limitations

## Security Best Practices

### Protecting Your Bot

1. **Secure your credentials** - Never share or commit your `.env` file
2. **Use strong tokens** - Generate secure bot tokens from BotFather
3. **Limit access** - Control who can interact with your bot
4. **Monitor usage** - Review logs for suspicious activity
5. **Update regularly** - Keep dependencies and system packages current
6. **Run with minimal privileges** - Use dedicated user accounts

### Data Protection

1. **Reports contain sensitive information** - Store securely
2. **Delete reports after use** - Don't accumulate vulnerability data
3. **Encrypt sensitive data** - If storing reports long-term
4. **Control access** - Limit who can view scan results
5. **Comply with data protection laws** - GDPR, CCPA, etc.

## Bug Bounty Programs

1. **Read the program rules carefully** - Each program has specific terms
